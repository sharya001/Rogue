“工匠工坊”是肉鸽游戏从“好玩”走向“让人上瘾”的关键。它解决了玩家死亡后的挫败感，将“失败”转化为了“积累”。

结合你现有的单文件 HTML 项目，我为你设计了一套完整的工匠工坊实现方案：

### 💰 1. 元货币（通关点数）的设计
**核心逻辑：** 即使没有通关，只要玩过就能获得点数，但通关有额外奖励。
*   **获取公式建议**：`本次获得点数 = (到达层数 × 2) + (击杀怪物数 × 0.5) + (通关额外奖励: 50)`
*   **数据存储**：在 `localStorage` 中增加一个字段，例如 `metaPoints`。
    ```javascript
    // 伪代码：游戏结束结算时调用
    function calculateMetaPoints(floor, kills, isWin) {
        let points = floor * 2 + Math.floor(kills / 2);
        if (isWin) points += 50;
        return points;
    }
    ```

### 🌳 2. 永久升级树（数值与机制的平衡）
为了避免后期数值崩坏，建议将升级分为10个层级，每个层级设定解锁条件或价格上限：

| 升级项 | 类型 | 效果描述 | 单次花费 | 最大等级 |
| :--- | :--- | :--- | :--- | :--- |
| **铁匠的锤子** | 基础属性 | 增加初始生命值  | 递增 (如 10, 20, 30...) | 10级 |
| **幸运硬币** | 经济系统 | 商店物品价格降低  | 固定2%  | 10级 |
| **极速** | 增加移动速度 | 增加移动速度，初始为0 | 固定+1%  | 10级 |
| **贪婪** | 经济系统 | 增加掉落金币率 | 固定+2% | 10级 |
| **强攻** | 攻击系统 | 增加攻击力 | 固定+2 | 10级 |
| **防御** | 防御系统 | 增加防御力 | 固定+1 | 10级 |



### 🎨 3. 全新 UI 面板设计
由于是 HTML/CSS 开发，UI 面板可以采用经典的 **“三栏式布局”**，配合模态框（Modal）呈现：

*   **顶部资源栏**：左侧显示当前拥有的“工匠点数”，右侧可以放一个“重置进度”的隐藏按钮（给想重新挑战高难度的硬核玩家）。
*   **左侧导航区**：使用垂直选项卡（Tabs），比如 `[属性强化]`、`[探索道具]`、`[解锁内容]`。点击不同 Tab 切换中间的升级列表。
*   **中间升级列表区**：
    *   每一个升级项做成一个小卡片。
    *   **状态展示**：如果未解锁，显示灰色和锁头图标；如果已满级，显示“MAX”字样；如果可购买，高亮显示价格和“升级”按钮。
    *   **交互反馈**：鼠标悬停在升级项上时，弹出一个 Tooltip（提示框），详细说明这个升级的具体收益。
*   **右侧预览区（可选）**：当玩家点击某个升级项时，右侧显示该升级的精美图标或简短描述，增强视觉吸引力。

### ⚙️ 4. 技术落地指南（如何接入现有项目）

拆分文件，这里给出一个清晰的模块划分建议：

1.  **新建数据模块 (`src/js/09-meta.js`)**：
    *   专门负责处理 `localStorage` 中关于工匠点数的读取、保存和计算。
    *   提供类似 `MetaSystem.addPoints()` 和 `MetaSystem.getUpgrades()` 的全局方法供其他模块调用。

2.  **新建 UI 模块 (`src/js/10-workshop-ui.js`)**：
    *   负责渲染工匠工坊的 DOM 结构。
    *   编写点击升级按钮时的逻辑：先判断点数是否足够 -> 扣除点数 -> 更新本地存储 -> 刷新 UI 界面。

3.  **样式隔离 (`src/css/workshop.css`)**：
    *   为工匠工坊单独写一套 CSS，避免和游戏内的战斗 UI 发生样式冲突。可以使用特定的类名前缀，如 `.workshop-panel`。

4.  **入口绑定**：
    *   在你之前规划的“主菜单”中，添加一个【工匠工坊】按钮，点击后隐藏主菜单，显示工匠工坊的面板。

**💡 进阶小建议：**
为了让玩家更有成就感，你可以在主界面上加一句动态标语。比如当玩家购买了“铁匠的锤子”第1级后，主界面的标语从“欢迎来到深渊”变成“你的剑刃已微微发亮”。这种细节会让你的肉鸽游戏显得非常有质感！
